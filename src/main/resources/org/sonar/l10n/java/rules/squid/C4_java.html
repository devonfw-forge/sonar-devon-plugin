This rule checks that the service layer of one component does not depend on the logic layer of another component. Allowed dependencies from the service layer:

  • Logic layer of the same component
  • Common layer

Noncompliant Code Example

    package com.devonfw.ide.sonarqube.compA.service.api;

    import com.devonfw.ide.sonarqube.compB.logic.api.Bar;

    public class MyClass {}

Compliant Solution

    package com.devonfw.ide.sonarqube.compA.service.api;

    import com.devonfw.ide.sonarqube.compA.logic.api.Bar;

    public class MyClass {}
